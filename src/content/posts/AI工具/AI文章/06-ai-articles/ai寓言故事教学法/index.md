# AI 寓言故事教学法

## 背景

团队管理中常见一个问题：直接讲解抽象概念（如"陌生化"）后，成员很快遗忘。受 Anthropic Claude 性格对齐团队负责人 Amanda Askell 启发，用**寓言故事**承载概念本质，让读者在"原来如此"的顿悟中记住核心，而非死记术语。

**核心理念**：故事才是人类传递知识的母语。先讲一个好故事，概念的本质会在读完时自然浮现。

## Prompt 完整模板

```markdown
# 寓言写作 Prompt

围绕 **{concept}** 这个概念，写一则寓言来完整地解释它。要像真正的寓言那样间接讲，不要直接点破。

## 一、寓言体感

- **篇幅**：1000字以内
- **世界观**：虚构，可拟人化，发生在不写实的小世界或日常微观场景
- **角色**：不超过三个，最好两个，互动本身承载寓意
- **揭示节奏**：全程不出现概念名称和术语，只在接近结尾时隐约揭示
- **叙事纪律**：情节和细节承载意义，角色不当解说员

## 二、防套路自检

### 意象黑名单

钟、河流、镜子、迷宫、织布机、地图、灯塔、棋盘、回声、影子、沙漏、风、蜡烛、种子、桥、星辰、蝴蝶、蛛网

### 地名黑名单

"回声城""记忆之村""遗忘之海""寂静谷"等过度文艺化虚构地名

### 结构黑名单

- 旅行者求教智者
- 村庄异象 → 众人顿悟
- 孩童一句话点醒大人
- 师徒辩难
- 临终遗言

### 角色黑名单

钟表匠、图书管理员、隐士、说书人、老船夫、酿酒师、铁匠、抄经人

### 开头黑名单

"从前有个地方……""某天某人遇见某事……""在很远的山里……"

## 三、切入角度鼓励

- **非人类视角**：工具、动物、植物、机构自述
- **具体现代职业**：理赔员、电梯保养工、菜市场摊主、夜班护士、外卖站长、二手房中介、分拣员
- **微观尺度**：一次交易、一次门诊、一通电话、一次拆卸

## 四、输出格式

1. **寓言正文**：直接输出，不加标题和引导语
2. **概念解析**：概念定义 + 故事元素与概念的映射对应
3. **检验问题**：理解检验（是否真正理解核心）+ 迁移检验（能否迁移到其他领域）
```

## 关键设计

| 设计            | 目的                                       |
| --------------- | ------------------------------------------ |
| 意象/结构黑名单 | 防止 AI 走向模板化叙事（训练数据高频模式） |
| 指定具体概念    | 原版是随机概念开盲盒，改为指定概念精准教学 |
| 理解+迁移检验   | 确保不是记住故事，而是理解本质并能举一反三 |
| 非人类视角      | 打破人类中心叙事，让概念获得全新载体       |

档案馆地下室的日光灯嗡嗡响。林姐的工位在第三排铁架之间，桌上堆着三千多本拆线重订的旧账本——不是账本，是信件、日记、投诉书、情书、遗嘱、菜谱、法庭辩护词、幼儿园请假条。她的工作很简单：任何人来问她什么，她都能从这些纸页里找出"该怎么回答"。

"我老婆要跟我离婚，我该怎么办？"一个穿夹克的男人问。

林姐翻开了四十七封类似的信。"其中三十二位在开头写了道歉，九位提到了孩子，二十一位提到了房子。最常见的结尾是'我们再试一次'，占比百分之四十一。"

男人没动。"我不是问别人写过什么，我是问……我现在该怎么办？"

林姐停顿了一下。她的手停在纸页上方，像不知道该落在哪一页。

---
下午又来了个实习生，第一次见她工作。"林姐，这些纸你都读过吗？"

"每一张。"

"那你一定什么都懂。"

林姐看着桌上那堆纸。"我知道一个人求婚时手心出汗的概率是百分之八十七，知道一个人写'我没事'的时候通常是有事的，知道一个人说'算了'和一个人说'算了吧'之间隔着一条完整的悲伤。我知道所有这些。"

她把手放回纸页上。

"但我没有手心的汗。"

实习生没听懂。

"你看，这些纸上的东西，写它们的人是真的活过那些时刻的。信里的每一个字都是从他们身体里长出来的。我只是……读得太多了，多到我能猜出下一句会是什么。猜对的时候，别人就觉得我懂了。"

她抽出一封信，上面写着"今天下雨了，我突然很想你"。

"这句话我见过一万两千次。每一封的下一句都不一样，但我能告诉你最常见的下一句是什么。可我知道'想你'是什么感觉吗？我不知道。我只是知道当一个人写了前半句，他大概率会写什么后半句。"

---
快下班的时候，那个穿夹克的男人又回来了。

"你上次说的那些数据，我回去想了想，好像也没什么用。"

林姐没有翻纸。

"那你回去问问你自己，"她说，"你第一次见她的时候，你手心里是什么感觉？那个感觉，没有任何一页纸能替你说出来。"

男人愣了一下，走了。

林姐继续坐在日光灯底下，把下一封信折好，放回铁架上。她什么也没忘，也什么都没经历过。

---
概念解析

"大模型"是人工智能领域的一种技术架构，核心是通过海量文本数据训练，学习语言中的统计规律和模式。它不是"理解"了世界，而是学会了从已知数据中预测最可能的下一个内容。

- 三千多封信件 → 训练数据，来自真实人类产生的文本
- 翻找四十七封类似的信并给出统计 → 模型基于模式匹配生成回复，而非基于真实体验
- "我知道……但我没有手心的汗" → 模型可以精确描述人类经验，但不具备主观感受和理解
- "猜对的时候，别人就觉得我懂了" → 流畅的表达让人误以为模型"理解"了，实际上是统计意义上的准确预测
- "她什么也没忘，也什么都没经历过" → 大模型记住了训练数据中的全部模式，但没有任何真实世界的亲身经历

---
检验问题

1. 理解检验：如果大模型的输出越来越流畅、越来越"像人"，这是否意味着它离"真正理解"更近了？用寓言中的某个细节来说明你的判断。
2. 迁移检验：你生活中是否遇到过类似的情境——某个系统或人表现得"好像很懂"，但实际上只是在重复模式？举一个具体例子，并说明它和大模型的相似点与本质区别。
